# 土壤墒情批量处理工具 - 使用说明

## 快速开始（无需部署）

### 步骤 1：下载文件

下载以下文件到同一目录：

1. **土壤墒情批量处理工具.zip**
   - 包含：`soil-moisture-hqt-1.0.0.jar`
   - 或单独下载 jar 文件

2. **运行脚本**
   - `运行.bat` - Windows 运行脚本

3. **配置文件**
   - `template.txt` - 文本模板
   - `config.json` - 替换规则配置

### 步骤 2：准备 Excel 文件

创建 `input/` 文件夹，放入 Excel 文件：

```
工具目录/
├── 运行.bat
├── soil-moisture-hqt-1.0.0.jar
├── template.txt
├── config.json
└── input/
    ├── 安康 2026.4.20 墒情.xlsx
    ├── 安康 2026.4.30 墒情.xlsx
    └── 安康 2026.5.10 墒情.xlsx
```

### 步骤 3：运行处理

双击 **`运行.bat`**

自动完成：
- ✅ 检查必要文件
- ✅ 批量处理所有 Excel
- ✅ 生成 `output/` 目录（独立文件）
- ✅ 生成 `merged.HQT`（合并文件）

## 自定义配置

### 修改 template.txt

编辑占位符格式：
```txt
618A0900，石泉，1,1,{date},08:00:00,105,...
```

### 修改 config.json

编辑替换规则：
```json
[
  {
    "row": 2,
    "col": 1,
    "regexPattern": "^((?:[^,]*,){4})[^,]+"
  }
]
```

- `row`: Excel 行号（从 1 开始）
- `col`: Excel 列号（从 1 开始）
- `regexPattern`: 匹配模式

## 常见问题

### Q: 提示找不到 Java

**解决**：安装 JDK 17+ 并配置环境变量

### Q: 没有输出文件

**检查**：
1. `input/` 目录是否有 Excel 文件
2. `template.txt` 和 `config.json` 是否存在
3. 查看命令行错误信息

### Q: 中文乱码

**解决**：确保文件使用 UTF-8 编码保存
